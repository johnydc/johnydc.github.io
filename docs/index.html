Detroit Tigers Home Run Predictor

The 2025 Detroit Tigers demonstrated the potential of a young club by finishing with the 11th most home runs in franchise history and making it to the American League Divisional Series. In order to forecast each player's home run totals for the 2026 season, this project creates a machine learning model. Along with more advanced analytics like bat speed, launch angle, and exit velocity, the model also includes conventional statistics like home runs, batting average, and plate appearances. Predictions for 13 qualifying players on the 2026 roster were produced using these features, showing the value of integrating traditional and advanced analytics for player forecasting and offering insights into possible offensive performance.

Problem

With the 11th-highest home run total in team history and a trip to the American League Divisional Series, the 2025 Detroit Tigers had an incredible season. The performance demonstrated the skill and potential of one of the league's youngest rosters, even though the team did not win the World Series. Accurately estimating individual contributions can assist in predicting team success and guide strategic choices, as many of the same players will be back in 2026.

Approach

The goal of this research is to apply machine learning to forecast the Detroit Tigers' home run totals in 2026. The dataset I used came from the MLB’s Baseball Savant website that contains a wide variety of every individual player’s batting analytics. The model I built combines advanced analytics like bat speed, launch angle, and exit velocity with conventional hitting statistics like home runs, batting averages, and plate appearances. The algorithm attempts to generate more precise predictions of individual offensive production by fusing player-specific metrics with previous performance data. Thirteen qualifying players on the 2026 roster were predicted, providing information about the team's possible performance and demonstrating the importance of data-driven methods in sports analytics.

Results & Impact

The model performed fairly well when predicting the home runs for the validation set. In its best run, the model only incurred about an 85 mean squared error across 36 samples. However, the model failed to predict any single digit home run totals for anyone, leading to a very biased model. Since the project is predicting the upcoming home runs totals for the Detroit Tigers, there is not an accurate way to measure the quality of the model in terms of the testing data. But from the perspective of a fan who has been following the Detroit Tigers for the past decade, I would conclude that these predictions are optimistic, but overall reasonable.

AWS Cloud Computing

I was able to access Amazon Web Services through a course I took at Michigan State University. Throughout that time, I explored the different services offered by AWS, and the overall architecture of the cloud. Additionally, I learned the benefits and drawbacks of using cloud computing. This project aims to demonstrates my basic understanding of AWS and cloud computing, as well as to compare the benefits and drawbacks of using cloud computing for machine learning tasks.

Problem

How much will cloud computing help with machine learning tasks? It is no secret that cloud computing is computationally intensive; but with that being said, it can also provide significant advantages. What are the benefits and drawbacks of using cloud computing for machine learning tasks?

Approach

I had the opportunity to work with AWS Sagemaker AI, which is a powerful tool for building, training, and deploying machine learning models at scale. I used Sagemaker to train a linear learner model on the NYC 311 dataset, which contains information about non-emergency service requests in New York City. The goal of the project was to predict the likelihood of a service request being closed within a certain time frame based on various features such as the type of request, location, and time of day. Through this project, I gained hands-on experience with AWS Sagemaker and learned how to leverage cloud computing for machine learning tasks. This projected explored the balance between the benefits of cloud computing, such as scalability and flexibility, and the drawbacks, such as cost and unnecessary complexity for smaller projects.

Results & Impact

In the end, I was able to successfully train a linear learner model on the NYC 311 dataset using AWS Sagemaker. The model was then compared to a simple logistic regression model trained on the same dataset using the sklearn library. The results achieved by both models were similar across various evaluation metrics, such as accuracy and F1 score. Given the lower complexity of this problem and the minimal improvements achieved with SageMaker, it appears unnecessary to implement a SageMaker workflow for this problem, considering the additional cost and complexity it introduces. However, for larger and more complex problems, the benefits of cloud computing may outweigh the drawbacks, making it a valuable tool for machine learning tasks.

Tracking Activity in Older Adults Using Machine Learning

According to the 2023 U.S. Census Bureau, about 28% of people 65 and older live by themselves. While this may seem common, it raises concerns regarding the safety and security of older adults who might not receive regular supervision. Physical strength and mobility naturally decline with age, making daily tasks more difficult. These changes can be subtle and often go unnoticed without regular monitoring. This project aims to address this issue by taking motion data from a batteryless motion sensor attached to an older adult and attempting to classify their daily activities. I plan to build a Recurrent Neural Network (RNN) model that takes in the time series data and outputs the corresponding activity. By developing a model that can recognize and track activity patterns in real time, monitoring the activity of older adults living independently becomes easier and more convenient. The expected outcome is to build a machine learning model capable of doing this task accurately, serving as a baseline system for healthcare providers and caregivers to detect unusual activity patterns without being physically present.

Problem

A growing number of older adults are living by themselves. This creates potential risks if the individual is not receiving regular supervision. The immediate concern of most caregivers would be potential falls and injuries. Another concern is increased inactivity, which can weaken the body over time. Healthcare providers and family members would benefit from a system that monitors daily activities without having to be physically present. Currently, doctors and family members receive information by scheduling appointments and hearing from the individual. This raises a potential problem if the information shared is inaccurate. Receiving additional unbiased data could be crucial in detecting lifestyle changes that might otherwise go unnoticed.

Approach

Throughout this project, I was able to build an end-to-end machine learning pipeline that provided valuable insights into the field of machine learning. I began with a basic logistic regression model and progressed to a Long Short-Term Memory (LSTM) network capable of processing sequential data and making predictions based on temporal patterns. This progression highlighted the power and flexibility of deep learning models. The most effective models were the Recurrent Neural Network (RNN) and the LSTM, due to their ability to capture sequential dependencies.

Results & Impact

The primary goal of this project was to successfully train a sequential LSTM model and improve classifier performance. While this goal was largely achieved, performance for class 4—corresponding to patient movement—did not improve as much as expected. Machine learning is a powerful tool, and when used correctly, can have profound effects. There is so much that goes into machine learning development, so it is extremely important to keep your workflow organized and methodical. This includes careful data preprocessing, thoughtful feature selection, proper model selection, and rigorous evaluation. Addressing each step thoroughly helps ensure that models are accurate, interpretable, and generalizable, ultimately leading to more reliable and impactful results.
